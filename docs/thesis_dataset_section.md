# Готовый текст для раздела НИРС про датасет

В рамках НИРС основной набор данных формируется не из готового внешнего корпуса, а с помощью контролируемого синтетического pipeline. Такой подход выбран по причине отсутствия удобного открытого русскоязычного датасета мошеннических телефонных и диалоговых сценариев, пригодного для воспроизводимого исследования. Поэтому центральным источником данных выступает внутренний synthetic core, создаваемый по схеме `LLM-generator -> LLM-augmentator -> LLM-validator / manual review -> final dataset`.

На стадии генерации используются заранее подготовленные seed-брифы, в которых фиксируются метка примера, тип сценария, обязательные и дополнительные признаки мошенничества, ограничения на нежелательные сигналы и уровень сложности. Это позволяет избежать хаотичной генерации и сделать датасет управляемым с точки зрения покрытия сценариев и баланса классов.

После генерации часть примеров проходит стадию аугментации. В работе рассматриваются переформулировки, скрытые hard-варианты и ASR-подобный шум. Такая аугментация нужна не только для увеличения объема данных, но и для отдельного эксперимента на устойчивость классификатора к перефразированию и ошибкам распознавания речи.

Далее каждый пример проходит валидацию. На этой стадии проверяется соответствие текста назначенной метке, наличие заявленных признаков, отсутствие персональных данных и внутренняя логическая согласованность сценария. Валидация может выполняться как дополнительным LLM-валидатором, так и ручной проверкой исследователя.

Целевой баланс основного synthetic core в НИРС составляет 40 примеров класса `fraud`, 20 примеров класса `suspicious` и 30 примеров класса `safe`. Именно этот набор используется как основа для расчета accuracy, precision, recall и F1-score, а также для сравнения rule-based baseline и LLM-подходов.

Наряду с внутренним synthetic core в работе используется внешний дополнительный набор данных на основе Kaggle-источника. Этот набор не рассматривается как основной вклад работы; он используется как независимый benchmark для дополнительного тестирования и обсуждения переносимости пайплайна на другой корпус.
