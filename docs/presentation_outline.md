# Презентация НИРС

## Слайд 1. Тема

LLM-пайплайн для выявления признаков мошенничества в текстовых и аудиотранскрибированных диалогах.

## Слайд 2. Актуальность

- Рост телефонного и мессенджер-мошенничества.
- Социальная инженерия часто обходится без вредоносного ПО.
- Простые keyword-правила плохо работают на переформулировках.

## Слайд 3. Проблема

- Нет удобного открытого русскоязычного fraud-audio датасета.
- Реальные данные ограничены, чувствительны и шумны.
- Нужен исследовательский baseline, а не production-система.

## Слайд 4. Цель и задачи

- Построить baseline-пайплайн text/audio -> ASR -> LLM analysis -> class.
- Сформировать синтетический русскоязычный датасет.
- Сравнить несколько архитектур анализа.

## Слайд 5. Данные

- Основной synthetic RU dataset.
- Safe-подмножество бытовых диалогов.
- Аугментации: paraphrase, subtle, ASR-noise.
- Небольшой real-audio subset для качественной валидации.

## Слайд 6. Таксономия сигналов

- urgency
- pressure
- bank/government/police impersonation
- sms_code_request / password_request / card_data_request
- safe_account_transfer
- secrecy / do_not_hang_up
- remote_access_app / phishing_link

## Слайд 7. Архитектуры

- rules baseline
- single LLM
- LLM + checklist
- LLM + self-check
- LLM ensemble

## Слайд 8. Эксперименты

- Базовая классификация.
- Устойчивость к аугментациям.
- Влияние ASR-ошибок.
- Качественная проверка на реальных аудио.

## Слайд 9. Результаты

Вставить таблицу из `outputs/metrics.csv` и 2-3 примера корректных/ошибочных классификаций.

## Слайд 10. Ограничения

- Синтетический датасет доминирует над real-world subset.
- Текущая версия baseline не использует fine-tuning.
- ASR-ошибки могут скрывать ключевые сигналы.

## Слайд 11. Что пойдет в диплом

- Подключение реальной LLM и/или локальной модели.
- Масштабирование датасета.
- Adversarial rewriting и prompt-injection robustness.
- Более сильная аудиочасть.

## Слайд 12. Вывод

Даже baseline-пайплайн можно оформить как полноценное исследование: данные, таксономия, архитектуры, метрики, ограничения, план развития.
