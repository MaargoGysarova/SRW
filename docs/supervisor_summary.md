# Короткое объяснение руководителю

В НИРС я строю baseline-пайплайн для выявления мошенничества в русскоязычных диалогах с помощью LLM-подхода. Основной эксперимент проводится на контролируемом синтетическом текстовом датасете, потому что открытых русских fraud-call датасетов практически нет.

В рамках работы я:

- формирую синтетический датасет `fraud / suspicious / safe`;
- задаю taxonomy сценариев и checklist сигналов мошенничества;
- сравниваю несколько архитектур: rule-based, single LLM, checklist, self-check и ensemble;
- отдельно проверяю устойчивость к переформулировкам и ASR-подобному шуму;
- использую аудиочасть как дополнительный pipeline `audio -> ASR -> LLM`, а не как основной эксперимент.

Таким образом, работа выглядит как исследовательский baseline с воспроизводимым кодом, метриками, ограничениями и понятным переходом к дипломной работе.
