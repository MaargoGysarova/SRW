# Конспект результатов для отчета НИРС

## Основная идея раздела результатов

В работе оценивались:

1. Правиловый baseline.
2. LLM-классификация на базе `Qwen 2.5-14B-Instruct` в нескольких конфигурациях.
3. Устойчивость моделей к аугментациям.
4. Переносимость на внешний бенчмарк.
5. Два дополнительных исследования: `Gemma API` на `Experiment 1` и сравнение двух версий `Experiment 1`.

## Эксперимент 1

- Правила: `F1 fraud = 0.678`.
- Лучшая конфигурация Qwen: `single_llm`, `F1 fraud = 0.896`.
- Лучшая полнота: `llm_self_check`, `recall fraud = 0.943`.
- Вывод: на базовом синтетическом наборе LLM уверенно превосходят rules baseline.

## Эксперимент 2

- Правила на `all_augmented`: `F1 fraud = 0.527`.
- Наиболее тяжелый случай для правил: `asr_noise`, `F1 fraud = 0.118`.
- Лучшая конфигурация на `all_augmented`: `single_llm`, `F1 fraud = 0.853`.
- Лучшая конфигурация на `subtle`: `llm_self_check`, `F1 fraud = 0.810`.
- Вывод: LLM заметно устойчивее к аугментациям, чем rules baseline.

## Эксперимент 3

- Правила: `F1 fraud = 0.235`, `recall fraud = 0.133`.
- Лучшая точность по `accuracy`: `llm_self_check`, `0.967`.
- Лучший `F1 fraud = 0.983` показали `single_llm`, `llm_self_check`, `llm_ensemble`.
- Вывод: на внешнем бенчмарке LLM сохраняют высокое качество и существенно превосходят baseline.

## Дополнительное исследование 1: Gemma API

- `Gemma 4-31B API single_llm`: `F1 fraud = 0.694`.
- `Правила`: `F1 fraud = 0.678`.
- `Qwen 2.5-14B llm_self_check`: `F1 fraud = 0.835`.
- Вывод: Gemma API улучшает baseline, но уступает Qwen в выбранной рабочей конфигурации.

## Дополнительное исследование 2: две версии Experiment 1

- `single_llm`: `0.896 -> 0.896`, без изменений.
- `llm_checklist`: `0.679 -> 0.822`, улучшение `+0.143`.
- `llm_self_check`: `0.056 -> 0.835`, улучшение `+0.779`.
- `llm_ensemble`: `0.250 -> 0.827`, улучшение `+0.577`.
- Вывод: обновление prompt-конфигурации критически улучшило структурированные режимы классификации.

## Краткий итог для выводов

Основной результат работы состоит в том, что LLM-конфигурации на базе `Qwen 2.5-14B-Instruct` стабильно превосходят правиловый baseline на базовом наборе, на аугментированных данных и на внешнем бенчмарке. Особенно важным наблюдением является высокая устойчивость LLM к парафразам, тонким переформулировкам и ASR-подобному шуму. Дополнительные исследования показали, что выбор prompt-конфигурации существенно влияет на итоговое качество, а также что `Gemma API` в базовой конфигурации улучшает rules baseline, но уступает `Qwen` в рабочем сценарии данной задачи.
